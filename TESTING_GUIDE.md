# 功能测试与验收指南

## ⚠️ 前置条件
**必须在 MemFire/Supabase 控制台执行 `COMPLETE_DB_SETUP.sql` 中的脚本**，否则以下功能将报错：
1. 自动生成头像/昵称
2. 点赞功能
3. 评论功能
4. 帖子列表获取

---

## 🧪 测试流程清单

### 1. 登录与自动建档
- [ ] **场景**：使用未注册过的邮箱登录（或直接使用“邮箱密码”注册）。
- [ ] **验证**：
  - 登录成功后，页面右上角显示“欢迎回来”。
  - 此时数据库 `profiles` 表应自动新增一条记录（由触发器生成）。
  - 发布框提示语变为“分享你的想法...”。

### 2. 发布帖子
- [ ] **场景**：在输入框输入文本（如“测试发布功能”），点击发布。
- [ ] **验证**：
  - 帖子立即出现在列表顶部。
  - **关键点**：头像和昵称应正常显示（不是“匿名用户”），这证明自动建档触发器生效了。

### 3. 点赞互动
- [ ] **场景**：点击帖子底部的“爱心”图标。
- [ ] **验证**：
  - 图标变为红色实心。
  - 数字立即 +1。
  - 刷新页面，状态保持不变（证明已写入数据库）。
  - 再次点击，图标变回空心，数字 -1。

### 4. 评论功能
- [ ] **场景**：点击“评论”按钮，展开评论区。
- [ ] **验证**：
  - 输入评论内容并发送。
  - 评论立即显示在下方。
  - 评论包含正确的用户名和时间。
  - 切换另一个浏览器/无痕模式（不登录），可以看到该评论但无法发表。

### 5. 角落 AI 助手
- [ ] **场景**：点击右下角机器人图标。
- [ ] **验证**：
  - 发送“你好”，能收到回复。
  - 确认回复内容符合预期（基于你配置的 DashScope/OpenAI 模型）。

---

## 🛠 故障排查

| 现象 | 可能原因 | 解决方案 |
|------|----------|----------|
| **发布后显示“匿名用户”** | 数据库触发器未创建 | 在控制台执行 `COMPLETE_DB_SETUP.sql` 中的脚本 |
| **点赞/评论没反应或报错** | 表不存在或 RLS 策略未配 | 在控制台执行 `COMPLETE_DB_SETUP.sql` 中的脚本 |
| **AI 助手不回复** | 环境变量未生效 | 检查 `.env.local` 中的 API Key，重启开发服务器 |
